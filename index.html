Principles
開発原則

意思決定・設計・レビュー時に立ち返る、チームの羅針盤です。

01
シンプルさを優先する
複雑な解決策より、単純で明快な解決策を選ぶ。「今必要なもの」だけを作る。
02
セキュリティはオプションではない
セキュリティ対策は後付けではなく、設計の最初から組み込む。
03
透明性・記録を大切にする
口頭で決めたことは記録に残す。「なぜそうしたか」の文脈を守る。
04
小さく動かして確認する
大きな変更を一度に入れない。小さい PR・小さいリリースを繰り返す。
05
レビューは品質への投資
コードレビューは「設計の意図・影響範囲・代替案」まで考える場。
06
自動化できることは自動化する
テスト・Lint・セキュリティスキャンは CI/CD で自動化し、人間はより重要な判断に集中する。
07
FD-E と FD-C は対等なパートナー
技術と顧客・事業観点は上下関係ではなく、相互補完の関係。
How to Join
参加申請の方法

参加申請は Pull Request で行います。以下の手順に沿って進めてください。
PR がマージされると、TED-Administrator から Organization への招待が届きます。

1

このリポジトリを Fork する

GitHub の「Fork」ボタンから、自分のアカウントにコピーを作成してください。

2

自己紹介ファイルを作成する

members/ ディレクトリに your-github-username.md というファイルを作成してください。

members/your-github-username.md
3

テンプレートに沿って記入する

members/_template.md の内容をコピーして、自己紹介を記入してください。

4

Pull Request を送る

Fork 元（ted-occ/web-welcome）の main ブランチに向けて PR を作成してください。PR タイトルは以下の形式でお願いします。

Join request: @your-github-username
5

招待を待つ

TED-Administrator が PR を確認し、問題がなければマージ後に Organization への招待を送ります。GitHub 登録メールアドレスに通知が届きます。
