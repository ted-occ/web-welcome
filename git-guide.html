Section 01

Git とは？ GitHub とは？

まずは、参加申請で使う「Git」と「GitHub」がそれぞれ何なのかを理解しましょう。難しく考える必要はありません。

Git

ファイルの変更履歴を記録・管理するための仕組みです。「いつ、誰が、何を変更したか」がすべて記録されるので、チームで安全にファイルを編集できます。

GitHub

Git で管理されたファイルをインターネット上で共有・共同編集するための Web サービスです。ブラウザだけで操作でき、今回の参加申請もブラウザだけで完了できます。

ポイント

ted-occ への参加申請は、ブラウザ上の GitHub だけで完結します。コマンドラインやソフトウェアのインストールは不要です。

参加申請で登場する主な概念を簡単に紹介します。

リポジトリ

プロジェクトのファイル一式とその変更履歴がまとまった「入れ物」のようなもの。

Fork（フォーク）

他の人のリポジトリを自分のアカウントにコピーすること。コピー先で自由に編集しても、元のリポジトリには影響しません。

コミット

ファイルの変更を「保存」すること。ゲームのセーブポイントのように、いつでもその時点の状態に戻れます。

Pull Request（PR）

自分が行った変更を、元のリポジトリに取り込んでもらうための「提案」。レビューを経て承認されるとマージ（統合）されます。

Section 02

GitHub アカウントを作成する

すでにアカウントをお持ちの方はこのセクションをスキップしてください。

1

GitHub にアクセスする

ブラウザで github.com を開きます。トップページに「Sign up」ボタンが表示されるのでクリックします。

2

メールアドレスを入力する

普段使っているメールアドレスを入力します。このアドレスに後ほど Organization への招待メールが届きます。

3

パスワードとユーザー名を設定する

ユーザー名はチーム内で識別に使われます。本名でなくても構いませんが、わかりやすい名前がおすすめです。参加申請のファイル名にもこのユーザー名を使います。

4

メール認証を完了する

入力したメールアドレスに確認コードが届きます。コードを入力してアカウント作成を完了させてください。

ヒント

GitHub アカウントは無料で作成できます。有料プランへのアップグレードは不要です。

Section 03

リポジトリを Fork する

Fork とは、ted-occ のリポジトリを自分のアカウントにコピーする操作です。コピー先で自由にファイルを追加・編集できます。

1

リポジトリページを開く

以下のリンクから ted-occ/web-welcome のリポジトリページにアクセスします。

https://github.com/ted-occ/web-welcome
2

「Fork」ボタンをクリックする

ページ右上にある 「Fork」 ボタンをクリックします。ボタンにはフォークのアイコン（）が付いています。

3

Fork 先を確認して作成する

「Create a new fork」画面が表示されます。Owner（オーナー）が自分のアカウントになっていることを確認して、「Create fork」 ボタンをクリックします。設定はそのままで大丈夫です。

4

Fork 完了を確認する

自分のアカウント配下にリポジトリがコピーされます。ページ左上のリポジトリ名が あなたのユーザー名/web-welcome になっていれば成功です。その下に「forked from ted-occ/web-welcome」と表示されます。

注意

Fork は元のリポジトリへの変更を提案するために行います。Fork 先で編集しても、元のリポジトリには自動で反映されません。変更を反映するには、次のステップで Pull Request を送ります。

Section 04

ファイルを作成・編集する

Fork したリポジトリで、自分の自己紹介ファイルを作成します。すべてブラウザ上で行えます。

1

members フォルダを開く

Fork したリポジトリのファイル一覧から members フォルダをクリックして開きます。

2

テンプレートの内容をコピーする

_template.md をクリックして開き、ファイルの内容をすべてコピーします。右上のコピーボタン（）を使うと簡単です。

3

新しいファイルを作成する

members フォルダに戻り、ページ右上の 「Add file」 > 「Create new file」 をクリックします。

ファイル名の入力欄に、自分のGitHubユーザー名を入力し、末尾に .md を付けます。

# ファイル名の例（ユーザー名が tanaka-taro の場合）
tanaka-taro.md
4

テンプレートを貼り付けて記入する

エディタ画面にコピーしたテンプレートの内容を貼り付け、各項目を記入します。

記入する項目

GitHub ユーザー名 — あなたの GitHub ユーザー名

自己紹介 — 簡単な自己紹介（経験年数や得意分野など）

希望するロール — FD-E（Engineering）か FD-C（Coordinating）にチェック

得意な技術・スキル — 使える言語やツールなど

参加動機 — なぜ ted-occ に参加したいか

連絡先（任意） — メールや SNS（書かなくても OK）

5

ファイルをコミットする

記入が終わったら、ページ右上の 「Commit changes...」 ボタンをクリックします。表示されるダイアログで、コミットメッセージはそのままでも構いません。「Commit changes」 をクリックして保存します。

ロールの選び方

FD-E（Engineering）は技術的な開発を担当するロール、FD-C（Coordinating）は顧客・事業面から開発を支えるロールです。どちらも対等なパートナーであり、上下関係はありません。自分の得意な方を選びましょう。

Section 05

Pull Request を送る

ファイルの作成が完了したら、元のリポジトリに「この変更を取り込んでください」という提案（Pull Request）を送ります。

1

Pull Request の作成画面を開く

Fork したリポジトリのトップページに戻ると、「This branch is 1 commit ahead of ...」というメッセージが表示されています。その右にある 「Contribute」 ボタンをクリックし、「Open pull request」 を選択します。

2

PR のタイトルを設定する

タイトル欄に以下の形式で入力します。@ の後にご自身の GitHub ユーザー名を記入してください。

Join request: @your-github-username
3

送信先を確認する

画面上部に「base repository: ted-occ/web-welcome」「base: main」と表示されていることを確認します。これが「どこに送るか」の指定です。

4

Pull Request を作成する

内容を確認したら 「Create pull request」 ボタンをクリックします。これで参加申請は完了です！

5

レビューと承認を待つ

TED-Administrator が PR を確認します。問題がなければマージされ、Organization への招待メールが GitHub に登録したメールアドレスに届きます。

PR 作成後のポイント

PR を送った後、レビュアーからコメントで修正を求められることがあります。その場合は、Fork したリポジトリ内の同じファイルを編集するだけで、PR に自動的に反映されます。

Section 06

用語集

このガイドや参加申請で出てくる用語をまとめました。

リポジトリ（Repository）

プロジェクトのファイルと変更履歴を保管する場所。GitHub 上で公開・共有できます。

Fork（フォーク）

他人のリポジトリを自分のアカウントにコピーすること。コピー先は独立しており、自由に編集できます。

コミット（Commit）

ファイルの変更内容を記録すること。「保存」に近い操作で、メッセージ付きで履歴に残ります。

ブランチ（Branch）

変更を分岐させる仕組み。今回は main ブランチだけ意識すれば大丈夫です。

Pull Request（PR）

自分の変更を元のリポジトリに取り込んでもらうための提案。レビューを経て承認・マージされます。

マージ（Merge）

Pull Request が承認され、変更が元のリポジトリに統合されること。参加申請の場合、マージ後に招待が届きます。
